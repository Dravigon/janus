Detailed Description

Mandatory interface for Phase 1.

All Janus performers should adhere to this interface.

Overview

A Janus application begins with a call to janus_initialize. New templates are constructed with janus_allocate and provided image data with janus_augment. Templates are freed after use with janus_free.

Templates can be used for verification with janus_flatten_template and janus_verify, or search with janus_enroll, janus_flatten_gallery and janus_search. Complete similarity matrix construction is offered by janus_compare.

Transfer learning is optionally offered by janus_train, whose model file output is then passed to janus_initialize.

All Janus applications end with a call to janus_finalize.

Thread Safety

All functions are marked one of:

  • thread-safe
    Can be called simultaneously from multiple threads, even when the invocations use shared data.
  • reentrant
    Can be called simultaneously from multiple threads, but only if each invocation uses its own data.
  • thread-unsafe
    Can not be called simultaneously from multiple threads.

Implementer Notes

  • Define JANUS_LIBRARY during compilation to export Janus symbols and compile a Unix implementation with -fvisibility=hidden.
  • Follow the Filesystem Hierarchy Standard by organizing the implementation into bin, include, lib, share and src sub-folders.

typedef struct janus_image janus_image

Common representation for still images and video frames.

Pixels are stored continuously in the janus_image::data buffer. Pixel layout with respect to decreasing spatial locality is channel, column, row. Thus pixel intensity can be retrieved as follows:

janus_data get_intensity(janus_image image, size_t channel, size_t column,
size_t row)
{
const size_t columnStep = (image.image_format == JANUS_COLOR ? 3 : 1);
const size_t rowStep = image.columns * columnStep;
const size_t index = row*rowStep + column*columnStep + channel;
return image.data[index];
}

(0, 0) corresponds to the top-left corner of the image. (width-1, height-1) corresponds to the bottom-right corner of the image.

janus_error janus_enroll ( const janus_template  template_,
const janus_template_id  template_id,
janus_gallery  gallery 
)

Add a template to the gallery.

Use janus_search for searching against the gallery.

It is up to the user to provide unique template_id values. The implementation may assume that multiple templates with the same template_id belong to the same identity.

The number of templates in the gallery can be retrieved by janus_gallery_size.

Parameters
[in]template_The template to add.
[in]template_idA unique identifier for the template.
[in]galleryThe gallery file to take ownership of the template. A new file will be created if one does not already exist.
Remarks
This function is reentrant.
