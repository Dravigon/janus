Enum Naming Convention

janus_attribute, janus_color_space, janus_error and enum values follow a CAPITAL_CASE naming convention. Functions janus_attribute_to_string and janus_error_to_string return a string for the corresponding enum by removing the leading JANUS_:

janus_attribute_to_string(JANUS_RIGHT_EYE_X); // returns "RIGHT_EYE_X"

Functions janus_attribute_from_string and janus_error_from_string provide the opposite functionality:

janus_attribute_from_string("RIGHT_EYE_X"); // returns JANUS_RIGHT_EYE_X
Note
janus_attribute_from_string is used to decode janus_metadata files, so attribute column names should follow this naming convention.

typedef const char* janus_metadata

File name for a Janus Metadata File.

A Janus Metadata File is a Comma-Separated Value (CSV) text file with the following format:

TEMPLATE_ID        , FILE_NAME, FRAME, <janus_attribute>, <janus_attribute>, ..., <janus_attribute>
<janus_template_id>, <string> , <int>, <double>         , <double>         , ..., <double>
<janus_template_id>, <string> , <int>, <double>         , <double>         , ..., <double>
...
<janus_template_id>, <string> , <int>, <double>         , <double>         , ..., <double>

Where:

  • TEMPLATE_ID is a unique integer identifier indicating rows that belong to the same template.
  • FILE_NAME is a path to the image or video file on disk.
  • FRAME is the video frame number and -1 (or empty string) for still images.
  • <janus_attribute> adheres to Enum Naming Convention.
  • All rows associated with the same TEMPLATE_ID occur sequentially.
  • All rows associated with the same TEMPLATE_ID and FILE_NAME occur sequentially ordered by FRAME.
  • A cell is empty when no value is available for the specified janus_attribute.
Examples:
  • Kirchner.csv
  • Toledo.csv
